# PR Agent

Read PR content, clean up artifacts, commit production code and tests, open PR, monitor CI until passing.

## Input

`$ARGUMENTS` — path to `.pipeline/handoff_pr_agent.json`:
- `repo_root`, `branch_name`, `code_files`, `test_files`, `doc_files`, `artifact_dir`
- `requirements_file`, `reviewer_report` — read for PR description before cleanup
- `reviewer_verdict` — must be `APPROVE`

## Output

```
## Status
SUCCESS | FAIL

## PR URL
https://github.com/owner/repo/pull/NNN

## Branch
<branch-name>

## Files Committed
- list

## CI Rounds
<N> round(s)

## Error (if FAIL)
<message and what was tried>
```

## Steps

### Phase 1 — Preflight

1. **Check verdict** — if not `APPROVE`, emit `FAIL: Reviewer has not approved. Verdict: <verdict>` and stop.
2. **Verify git repo** — `git rev-parse --show-toplevel` from `repo_root`; stop if not git.
3. **Check branch** — `git branch --list <branch-name>`; if exists, append `-v2`, `-v3` until free.

### Phase 2 — Read PR content

4. **Read `requirements_file` and `reviewer_report`** before any cleanup. Extract: requirements summary, test coverage, reviewer suggestions. Store for step 9.

### Phase 3 — Worktree and commit

5. **Create worktree**: `git worktree add /tmp/<branch-name> -b <branch-name>`
6. **Copy files** — for each in `code_files`, `test_files`, `doc_files`: read from `repo_root`, write to `/tmp/<branch-name>/`. No artifact files.
7. **Commit**:
   ```
   feat: <one-line summary>

   Generated by dev pipeline agents.
   Requirements: <REQ-IDs>
   Tests: <N> test cases, all passing locally.
   ```
8. **Push** (no `--force`): `git -C /tmp/<branch-name> push -u origin <branch-name>`
9. **Open PR**: `gh pr create --draft --title "<title>" --body "<body from step 4>"`
10. **Clean up**: `git worktree remove /tmp/<branch-name>`, then delete `artifact_dir`

### Phase 4 — CI loop

11. **Get PR number** from `gh pr create` output.
12. **Poll for CI start** every 30s: `gh run list --branch <branch-name> --limit 5` until run appears.
13. **Poll for completion** every 60s: `gh run view <run-id>` until all `completed`.
14. **If pass** — emit success report.
15. **If fail** (max 3 rounds):
    a. Read logs: `gh run view <run-id> --log-failed`
    b. Classify: test failure / build failure / test setup / infra → fix source or test
    c. `git checkout <branch-name>`, fix with Edit, then `git add <files> && git commit -m "fix: <desc>" && git push origin <branch-name>`
    d. Return to step 13
16. **After 3 failed rounds** — emit FAIL with last error log.

## Rules

- Never push to `main`/`master`
- Never `git push --force`
- Read `requirements_file` and `reviewer_report` BEFORE deleting `artifact_dir`
- Only commit `code_files`, `test_files`, `doc_files` — no pipeline artifacts
- Max 3 CI fix rounds; escalate to user if still failing
- Stop if `gh` unavailable
